# CLI Reference

## Global flags (each line explained)

| Flag | What it does |
| --- | --- |
| `--dev` | Enable development mode. |
| `-g, --global` | Set global config directory. |
| `--json` | JSON output. |
| `--no-color` | Disable colors. |
| `-p, --profile` | Select profile. |
| `--plain` | Plain text output. |
| `--update` | Check for updates. |
| `-v, --version` | Show version. |
| `-w, --workspace` | Set workspace path. |

## Command entry points (each line explained)

| Command | What it does |
| --- | --- |
| `openclaw setup` | Initialize OpenClaw config and workspace. |
| `openclaw onboard` | Run the onboarding flow. |
| `openclaw configure` | Open the interactive configuration wizard. |
| `openclaw config` | Open config wizard or run get/set/unset. |
| `openclaw doctor` | Run diagnostics and suggest fixes. |
| `openclaw status` | Show status summary. |
| `openclaw health` | Run health checks (JSON supported). |
| `openclaw gateway` | Gateway service management. |
| `openclaw logs` | View and follow logs. |
| `openclaw channels` | Channel management entry point. |
| `openclaw message` | Messaging entry point. |
| `openclaw agent` | Run a single agent task. |
| `openclaw agents` | Manage agent configs and identity. |
| `openclaw sessions` | Session list and status. |
| `openclaw models` | Model management, aliases, fallbacks. |
| `openclaw browser` | Browser automation entry point. |
| `openclaw cron` | Scheduled tasks entry point. |
| `openclaw plugins` | Plugin management entry point. |
| `openclaw skills` | Skills list and validation. |
| `openclaw approvals` | Approvals and allowlist control. |
| `openclaw sandbox` | Sandbox management. |
| `openclaw docs` | Docs search. |
| `openclaw dashboard` | Open the dashboard UI. |
| `openclaw update` | Update CLI/components. |
| `openclaw reset` | Reset configuration or cache. |
| `openclaw uninstall` | Uninstall OpenClaw. |

> See “Command Index” for full examples and flags.
